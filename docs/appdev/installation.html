설치

설치

톰캣을 웹 어플리케이션을 개발하는세 사용하려면, 먼저 그것을 설치해야 합니다(종속되는 소프트웨어 또한). 필요한 단계는 아래의 세부구문에 설명되어 있습니다.

JDK

톰캣 7.0은 Java SE 6위에서 돌아가게끔 설계 되어있습니다.

JDK와 호환되는 많은 플랫폼(또는 연결을 통해 찾을 수 있는 것들)에서도 가능합니다. http://www.oracle.com/technetwork/java/javase/downloads/index.html.

Tomcat

톰캣 서버 바이너리 다운로드는 http://tomcat.apache.org/에서 가능합니다. 이 메뉴얼은 당신이 가장 퇴근에 배포된 톰캣 7을 사용한다는 가정을 하고 있습니다. 톰캣의 다운로드와 설치에 대해 자세히 알고싶으시면 이곳 에서 가능합니다.

이 설명서의 나머지 부분의, 예제 쉘 스크립트는 당신이 톰캣이 설치된 디렉토리의 경로가 CATALINA_HOME 환경 변수에 포함되어있다는 것을 가정하고 있습니다. 선택적으로 만약 톰캣이 여러개 존재하는 것을 위해 설정이 되어있다면, 각각의 톰캣은 그것의 CATALINA_BASE 설정을 가지고 있을 것 입니다.

Ant

Ant 빌드 툴의 바이너리 다운로드는 http://ant.apache.org/에서 가능합니다. 이 메뉴얼은 당신이 Ant 1.8이나 그 이상을 쓴다고 가정합니다. 명령은 다른 버전과 호환되지만 테스트되지는 않았습니다.

CVS

위에서 설명한 필수 도구 외에도, 당신은 웹 어플리케이션을 만들 때 소스 파일의 역사적 버전을 유지시키기 위해 Concurrent Version System (CVS)과 같은 소스 코드 컨트롤시스템을 다운로드 받고 설치하도록 강하게 격려받을 것 입니다. 서버 외에도, 당신은 소스 코드 파일을 체크 아웃하고, 변경된 버전을 체크 인 하기 위해 적당한 클라이언트 툴이 필요할 것 입니다.

소스 코드 컨트롤 어플리케이션을 설치하는 것과 사용하는 것에 대한 자세한 명령은 이 메뉴얼의 범위를 넘어갑니다. 그러나, 많은 플랫폼을 위한 CVS 서버와 클라이언트 툴(문서와 함께)은 http://www.cvshome.org/에서 다운받으실 수 있습니다.
